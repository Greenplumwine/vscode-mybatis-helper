## 技术方案变更说明

**原方案：** 使用 JLS 直接通信获取类成员
**问题：** vscode-java 未暴露 `java.getClassMembers` 等命令，仅提供：
- `getDocumentSymbols`: 需 URI，不能直接查类名
- `getClasspaths`: 获取类路径，无成员信息
- `goToDefinition`: 跳转定义，非查询成员

**新方案：** 混合解析策略

```
获取类 User 的属性列表
       |
       ▼
┌─────────────────────────────────┐
│ 1. 查找源码文件                  │
│    src/main/java/.../User.java   │
└─────────────────────────────────┘
       │
   找到 │──────┬────────┐
       │      │        │
       ▼      ▼        ▼
┌──────────┐ ┌──────────┐ ┌────────────┐
│ 源码解析  │ │ 源码解析  │ │  javap     │
│ (正则)   │ │ (失败)   │ │ (class文件)│
└──────────┘ └──────────┘ └────────────┘
       │            │            │
       ▼            ▼            ▼
   成功返回      返回空       成功返回
   属性列表      列表         属性列表
```

整合三种解析方式，提供统一的属性获取接口：

1. 实现 `getObjectProperties` 主方法：
   ```typescript
   public async getObjectProperties(className: string): Promise<string[]> {
     // 1. 查内存缓存
     if (this.cache.has(className)) {
       return this.cache.get(className)!;
     }

     // 2. 尝试源码解析（最准确，有泛型信息）
     const sourceFile = await this.findSourceFile(className);
     if (sourceFile) {
       const props = await this.parseSourceFile(sourceFile);
       this.cache.set(className, props);
       return props;
     }

     // 3. 尝试 javap 解析（适用于 JAR 中的类）
     const classFile = await this.findClassFile(className);
     if (classFile) {
       const props = await this.parseWithJavap(classFile);
       this.cache.set(className, props);
       return props;
     }

     // 4. 都失败，返回空
     return [];
   }
   ```

2. 实现 LRU 缓存：
   ```typescript
   private typeCache = new LRUCache<string, string[]>({
     max: 500,  // 最多缓存 500 个类型
     ttl: 1000 * 60 * 10  // 10 分钟过期
   });
   ```

3. 并发请求合并：
   ```typescript
   private pendingRequests = new Map<string, Promise<string[]>>();

   // 相同类名的并发请求合并为一个
   if (this.pendingRequests.has(className)) {
     return this.pendingRequests.get(className)!;
   }
   ```

4. 泛型类型处理：
   - `List<User>` → 提取 `User`，递归获取 User 的属性
   - `Map<K,V>` → 不展开（太复杂）
   - 数组类型 `User[]` → 提取 `User`
