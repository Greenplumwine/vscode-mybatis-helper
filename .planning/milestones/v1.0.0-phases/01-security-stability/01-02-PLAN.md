# Plan 01-02: Implement Three-Tier Error Handling

## Objective
Implement comprehensive error handling with three-tier strategy: critical errors notify users, general errors are logged, and degradation operations use debug logging only.

## Background
Current code has many silent error catches that make debugging difficult:
```typescript
try {
    // operation
} catch (e) {
    // ignore error - user never knows
}
```

This plan implements the error handling strategy decided in discuss-phase.

- src/extension.ts (error handling entry points)

<action>
Create error classification and handling utilities:

1. Create `src/utils/errorHandler.ts`:
   ```typescript
   import * as vscode from 'vscode';
   import { logger } from './logger';

   export enum ErrorSeverity {
     CRITICAL = 'critical',  // User notification + log
     GENERAL = 'general',    // Log only
     DEBUG = 'debug'         // Debug log only
   }

   export interface ErrorContext {
     operation: string;
     file?: string;
     detail?: string;
   }

   export function handleError(
     error: unknown,
     severity: ErrorSeverity,
     context: ErrorContext
   ): void {
     const errorMessage = error instanceof Error ? error.message : String(error);
     const fullMessage = `[${context.operation}] ${errorMessage}`;

     switch (severity) {
       case ErrorSeverity.CRITICAL:
         // Show to user
         vscode.window.showErrorMessage(`MyBatis Helper: ${fullMessage}`);
         // Also log
         logger.error(fullMessage, context);
         break;

       case ErrorSeverity.GENERAL:
         // Log to output channel
         logger.warn(fullMessage, context);
         break;

       case ErrorSeverity.DEBUG:
         // Debug level only
         logger.debug(fullMessage, context);
         break;
     }
   }

   export function handleAsyncError<T>(
     promise: Promise<T>,
     severity: ErrorSeverity,
     context: ErrorContext
   ): Promise<T | undefined> {
     return promise.catch(error => {
       handleError(error, severity, context);
       return undefined;
     });
   }
   ```

2. Update `src/utils/index.ts` to export:
   ```typescript
   export { handleError, handleAsyncError, ErrorSeverity, ErrorContext } from './errorHandler';
   ```
</action>

<acceptance_criteria>

- `handleError` function routes errors appropriately based on severity

         const error = new Error('Debug error');
         handleError(error, ErrorSeverity.DEBUG, mockContext);

       });
     });
   });
