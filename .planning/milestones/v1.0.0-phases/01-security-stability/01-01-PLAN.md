# Plan 01-01: Fix Command Injection Vulnerabilities

## Objective
Replace all `execSync` with string interpolation to `execFileSync` with array arguments in `enterpriseConfigResolver.ts` to eliminate command injection vulnerabilities.

## Background
The codebase has 5 locations using `execSync` with string interpolation that are vulnerable to command injection:
- Line 869: `execSync(`jar tf "${jarPath}"`)`
- Line 888: `execSync(`javap -v -classpath "${jarPath}" "${className}"`)`
- Line 922: `execSync(`javap -v "${classPath}"`)`

Reference implementation exists in `classParsingWorker.ts` line 112 which uses safe `execFileSync` with array arguments.

## Task 2: Replace execSync in enterpriseConfigResolver.ts

<read_first>
- src/features/mapping/enterpriseConfigResolver.ts (full file)
- src/utils/pathSecurity.ts (new utility)
</read_first>

<action>
Replace all `execSync` usages with `execFileSync` array arguments:

1. Line 869: Replace `execSync(`jar tf "${jarPath}"`)` with:
   ```typescript
   const safeJarPath = sanitizeJarPath(jarPath);
   if (!safeJarPath) {
     throw new Error(`Invalid jar path: ${jarPath}`);
   }
   const output = execFileSync('jar', ['tf', safeJarPath], { encoding: 'utf-8' });
   ```

2. Line 888: Replace `execSync(`javap -v -classpath "${jarPath}" "${className}"`)` with:
   ```typescript
   const safeJarPath = sanitizeJarPath(jarPath);
   if (!safeJarPath) {
     throw new Error(`Invalid jar path: ${jarPath}`);
   }
   // className should be validated - ensure it matches pattern /^[\w.$]+$/
   if (!/^[\w.$]+$/.test(className)) {
     throw new Error(`Invalid class name: ${className}`);
   }
   const output = execFileSync('javap', ['-v', '-classpath', safeJarPath, className], { encoding: 'utf-8' });
   ```

3. Line 922: Replace `execSync(`javap -v "${classPath}"`)` with:
   ```typescript
   const safeClassPath = sanitizeClassPath(classPath);
   if (!safeClassPath) {
     throw new Error(`Invalid class path: ${classPath}`);
   }
   const output = execFileSync('javap', ['-v', safeClassPath], { encoding: 'utf-8', timeout: 2000 });
   ```

4. Add import at top of file:
   ```typescript
   import { execFileSync } from 'child_process';
   import { sanitizeJarPath, sanitizeClassPath } from '../../utils/pathSecurity';
   ```

5. Remove the dynamic `require('child_process')` calls since we're using static imports
</action>

<acceptance_criteria>
- All `execSync` calls removed from enterpriseConfigResolver.ts
- All `execFileSync` calls use array arguments (not string interpolation)
- All paths are validated with sanitize functions before use
- Class names are validated with regex `/^[\w.$]+$/` before use
- `pnpm run compile` passes with no errors

## Task 3: Add security tests

<read_first>
- src/features/mapping/enterpriseConfigResolver.ts (updated)
- src/utils/pathSecurity.ts
</read_first>

<action>
Create security-focused unit tests:

1. Create `src/utils/__tests__/pathSecurity.test.ts`:
   ```typescript
   import { sanitizeClassPath, sanitizeJarPath } from '../pathSecurity';
   import * as fs from 'fs';
   import * as path from 'path';

   describe('pathSecurity', () => {
     const testDir = path.join(__dirname, 'test-fixtures');

     beforeAll(() => {
       // Create test fixtures
       if (!fs.existsSync(testDir)) {
         fs.mkdirSync(testDir, { recursive: true });
       }
       fs.writeFileSync(path.join(testDir, 'Test.class'), 'fake class content');
       fs.writeFileSync(path.join(testDir, 'test.jar'), 'fake jar content');
     });

     afterAll(() => {
       // Cleanup
       fs.rmSync(testDir, { recursive: true, force: true });
     });

     describe('sanitizeClassPath', () => {
       it('should return absolute path for valid .class file', () => {
         const result = sanitizeClassPath(path.join(testDir, 'Test.class'));
         expect(result).toBe(path.join(testDir, 'Test.class'));
       });

       it('should return null for non-existent file', () => {
         const result = sanitizeClassPath(path.join(testDir, 'NonExistent.class'));
         expect(result).toBeNull();
       });

       it('should return null for wrong extension', () => {
         fs.writeFileSync(path.join(testDir, 'wrong.txt'), 'text');
         const result = sanitizeClassPath(path.join(testDir, 'wrong.txt'));
         expect(result).toBeNull();
         fs.unlinkSync(path.join(testDir, 'wrong.txt'));
       });

       it('should resolve relative paths', () => {
         const relativePath = path.relative(process.cwd(), path.join(testDir, 'Test.class'));
         const result = sanitizeClassPath(relativePath);
         expect(result).toBe(path.join(testDir, 'Test.class'));
       });
     });

     describe('sanitizeJarPath', () => {
       it('should return absolute path for valid .jar file', () => {
         const result = sanitizeJarPath(path.join(testDir, 'test.jar'));
         expect(result).toBe(path.join(testDir, 'test.jar'));
       });

       it('should return null for non-existent jar', () => {
         const result = sanitizeJarPath(path.join(testDir, 'nonexistent.jar'));
         expect(result).toBeNull();
       });

       it('should return null for .class file passed as jar', () => {
         const result = sanitizeJarPath(path.join(testDir, 'Test.class'));
         expect(result).toBeNull();
       });
     });
   });
   ```
