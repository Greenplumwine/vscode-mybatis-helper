# Plan 02-02: Async Operations Conversion

## Objective
Convert synchronous file operations to async in hot paths to prevent UI blocking during large project scans.

## Background
Current issues:
1. `fastScanner.ts` uses sync file operations in hot paths
2. `classFileWatcher.ts` uses sync `fs` operations and `execFileSync`
3. Large project scans can block the UI thread

Priority order (from CONTEXT.md D-02):
1. High: `fastScanner.ts` file traversal
2. Medium: `classFileWatcher.ts` monitoring
3. Low: Other auxiliary functions

---

## Task 1: Convert fastScanner.ts to async

<read_first>
- src/features/mapping/fastScanner.ts
- src/features/mapping/types.ts (for type definitions)
</read_first>

<action>
Convert sync file operations to async:

1. Replace `fs.*Sync` calls with `fs/promises` equivalents:
   - `fs.existsSync()` → `fs.access()` with try/catch
   - `fs.readFileSync()` → `fs.readFile()`
   - `fs.statSync()` → `fs.stat()`
   - `fs.readdirSync()` → `fs.readdir()`

2. Update method signatures to async:
   ```typescript
   // Before
   private scanXmlFiles(dir: string): XmlFileInfo[]

   // After
   private async scanXmlFiles(dir: string): Promise<XmlFileInfo[]>
   ```

3. Update batch processing to use async/await:
   ```typescript
   // Process files in parallel with limit
   private async processBatch<T, R>(
     items: T[],
     processor: (item: T) => Promise<R>,
     limit: number
   ): Promise<R[]> {
     const results: R[] = [];
     for (let i = 0; i < items.length; i += limit) {
       const batch = items.slice(i, i + limit);
       const batchResults = await Promise.all(batch.map(processor));
       results.push(...batchResults);
     }
     return results;
   }
   ```

4. Key methods to convert:
   - `scanXmlFiles()` → `scanXmlFilesAsync()`
   - `scanJavaFiles()` → `scanJavaFilesAsync()`
   - `parseXmlFile()` → `parseXmlFileAsync()`
   - `readFileContent()` → `readFileContentAsync()`

5. Update `scan()` method to use async operations:
   ```typescript
   public async scan(): Promise<void> {
     // ... existing setup ...

     // Phase 2: 并行扫描 XML 和 Java（使用 async）
     const [xmlFiles, javaFiles] = await Promise.all([
       this.scanXmlFilesAsync(xmlLocations),
       this.scanJavaFilesAsync(mapperScanConfigs)
     ]);

     // Phase 3: 异步解析 XML
     const xmlInfos = await this.processBatch(
       xmlFiles,
       file => this.parseXmlFileAsync(file),
       this.config.parallelLimit
     );

     // ... rest of scan logic ...
   }
   ```
</action>

<acceptance_criteria>
- All `fs.*Sync` calls replaced with async equivalents
- `scan()` method uses async file operations
- Batch processing uses parallel limit to control concurrency
- No UI blocking during large scans
- `pnpm run compile` passes
- `pnpm run test:unit` passes
</acceptance_criteria>

## Task 2: Convert classFileWatcher.ts to async

<read_first>
- src/features/mapping/classFileWatcher.ts
- src/utils/pathSecurity.ts (for execFileSync replacements)
</read_first>

<action>
Convert sync operations to async:

1. Replace `fs` (sync) with `fs/promises`:
   ```typescript
   // Before
   import * as fs from 'fs';

   // After
   import * as fs from 'fs/promises';
   ```

2. Convert file existence checks:
   ```typescript
   // Before
   if (fs.existsSync(filePath)) { ... }

   // After
   try {
     await fs.access(filePath);
     // file exists
   } catch {
     // file doesn't exist
   }
   ```

3. Convert file read operations:
   ```typescript
   // Before
   const content = fs.readFileSync(filePath, 'utf-8');

   // After
   const content = await fs.readFile(filePath, 'utf-8');
   ```

4. Replace `execFileSync` with `execFile` (async):
   ```typescript
   import { execFile } from 'child_process';
   import { promisify } from 'util';

   const execFileAsync = promisify(execFile);

   // Before
   const output = execFileSync('javap', ['-v', classPath], { encoding: 'utf-8' });

   // After
   const { stdout } = await execFileAsync('javap', ['-v', classPath], { encoding: 'utf-8' });
   ```

5. Update `handleClassFileChange` to be async:
   ```typescript
   private async handleClassFileChange(
     uri: vscode.Uri,
     type: 'create' | 'change' | 'delete'
   ): Promise<void> {
     // ... async implementation ...
   }
   ```

6. Update debounced handler to handle async:
   ```typescript
   private debouncedHandleChange(uri: vscode.Uri, type: 'create' | 'change' | 'delete'): void {
     if (this.debounceTimer) {
       clearTimeout(this.debounceTimer);
     }

     this.debounceTimer = setTimeout(async () => {
       try {
         await this.handleClassFileChange(uri, type);
       } catch (error) {
         this.logger?.error('Error handling class file change:', error);
       }
     }, this.DEBOUNCE_DELAY);
   }
   ```
</action>

<acceptance_criteria>
- `fs` (sync) replaced with `fs/promises`
- `execFileSync` replaced with async `execFile`
- `handleClassFileChange` is async
- Debounced handler properly awaits async operations
- Error handling preserved
- `pnpm run compile` passes
</acceptance_criteria>

## Task 3: Add progress reporting for long operations

<action>
Add VS Code progress notifications for long-running operations:

1. Update `scan()` method with progress reporting:
   ```typescript
   import * as vscode from 'vscode';

   public async scanWithProgress(): Promise<void> {
     await vscode.window.withProgress(
       {
         location: vscode.ProgressLocation.Notification,
         title: 'Scanning MyBatis mappers...',
         cancellable: true
       },
       async (progress, token) => {
         const startTime = Date.now();

         // Report progress during phases
         progress.report({ increment: 0, message: 'Finding XML files...' });
         const xmlFiles = await this.scanXmlFilesAsync(xmlLocations);

         if (token.isCancellationRequested) return;

         progress.report({ increment: 33, message: 'Finding Java files...' });
         const javaFiles = await this.scanJavaFilesAsync(mapperScanConfigs);

         if (token.isCancellationRequested) return;

         progress.report({ increment: 33, message: 'Building mappings...' });
         // ... rest of scan

         const duration = Date.now() - startTime;
         this.logger?.info(`Scan completed in ${duration}ms`);
       }
     );
   }
   ```

2. Add progress to `EnterpriseScanner` if applicable (check if it exists)
</action>

<acceptance_criteria>
- Progress notification shown during scan
- Progress increments at key phases
- Cancellation supported
- Progress automatically closes on completion
- `pnpm run compile` passes
</acceptance_criteria>
