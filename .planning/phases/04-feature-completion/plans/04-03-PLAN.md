   修改或扩展 SqlExtractor：
   ```typescript
   export class SqlExtractor implements PipelineStep {
     execute(content: string, context: PipelineContext): string

     // 新增：仅提取范围内的 SQL
   ```

3. 实现范围格式化逻辑：
   ```typescript
   async provideDocumentRangeFormattingEdits(
     document: vscode.TextDocument,
     range: vscode.Range,
     options: vscode.FormattingOptions,
     token: vscode.CancellationToken
   ): Promise<vscode.TextEdit[]> {
     // 1. 提取范围内的 SQL 区域
     const sqlRegions = this.extractSqlInRange(document, range);

     // 2. 格式化每个 SQL 区域
     const edits: vscode.TextEdit[] = [];
     for (const region of sqlRegions) {
       if (token.isCancellationRequested) break;

       const formatted = await this.formatSqlRegion(region, options);
       edits.push(new vscode.TextEdit(region.range, formatted));
     }

     return edits;
   }
   ```

4. 处理边界情况：
   - 范围跨越多个 SQL 语句：分别格式化
   - 范围在 SQL 语句中间：智能扩展到完整语句
   - 范围包含 XML 标签：只格式化标签内的 SQL

5. 保持与全文档格式化的兼容性：
   - 范围为空时退化为全文档格式化
   - 统一格式化选项处理
